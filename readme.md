# AI開発者向け　ニュースサマライズ

## 📝 目的
AIに関するニュースは日々目まぐるしくアップデートと新たな追加がなされています。
その中で、AIの開発担当者や開発のマネジメント・コンサルタントをしている人にとって重要な情報とそうでない情報が入り交ざっています。今回のAIでAI開発やマネジメント、コンサルティングをする人に向けた最新情報を定期的に更新し、わかりやすく届けるAPIを構築します。

## 🎯 要求・ゴール
* **自動収集**: 指定したサイトからAIに関する最新のニュースを自動収集。
* **分類とサマライズ**: AIのニュースをカテゴリ分けし、サマリーを作成
* **優先度決定**:　多くのニュースからAI開発担当やマネジメント担当の役に立つ情報のみをピックアップ 
* **要約文作成**: ピックアップした情報を整理し、文章として返却

## ⚙️ 要件定義
| 項目 | 内容 |
| :--- | :--- |
| **ユーザー入力** | 直近何日間のニュースを利用するか |
| **データソース** | AI関連のニュースを配信しているサイト |
| **収集ロジック** | データソースの特定された日数までのニュースをRSSを利用しピックアップ |
| **前処理** | データ定義の整理、統一 / 対象とするニュースの日数でフィルタリング |
| **LLMカテゴライズ** | LLMでタイトル情報からカテゴリ分けと優先順位付け |
| **情報をサマライズ** | URLへアクセスをする。情報をLLMでサマライズする |
| **出力形式** | JSON。サマリー、対象記事をカテゴリごとに提示 |

## 🏗️ システムアーキテクチャ
```text
ユーザー (APIクライアント)
  │
  │ 直近利用日数 (N日)
  ▼
[データ収集モジュール]
  │ RSSから指定期間の記事(タイトル/URL/概要)をバルク取得
  ▼
[前処理モジュール]
  │ フォーマット統一、重複排除
  ▼
[LLMカテゴライズ・優先度判定]
  │ タイトルからカテゴリ分類と優先順位(スコア)を決定
  ▼
[情報サマライズモジュール]
  │ 高優先度のURLへアクセスし、本文から重要事項を要約
  ▼
[出力モジュール]
    JSON形式で「カテゴリごとのサマリーと対象記事」を提示
```
